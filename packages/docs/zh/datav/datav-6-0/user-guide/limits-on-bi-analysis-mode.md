本文介绍BI分析模式在使用中的限制、产生变化时的操作及默认自动化配置等说明。

## 使用BI分析模式组件的限制须知

* 组件仅可使用DataV数据集作为数据源。
* 无法添加和使用数据过滤器。
* 当组件拥有多个数据接口时，仅其主数据接口可配置。

## 分析模式的操作变化说明

* 受到限制的操作

| 操作    | 变化说明                                                                                                                                                             |
| ---- | ---- |
| 图层移动  | 由于分析器内部有且仅有一个组件，因此将其它组件拖往分析器内部时的操作会失败并报错。同时，分析模式组件不可单独拖动，拖动后会提示是否需要连同分析器一并移动。                                                                                    |
| 复制和收藏 | 单独复制和收藏分析组件是不允许的，编辑器会提示是否需要连同分析器一并复制和收藏。                                                                                                                         |
| 导出到蓝图 | 分析模式组件无法导出到蓝图。此外，分析器可以导出到蓝图，详情请参见[分析器蓝图编辑器交互配置](https://help.aliyun.com/zh/datav/datav-6-0/user-guide/bi-analyzer-interaction-instructions#section-e70-vfl-jwm)。 |
| 转为区块  | 分析模式组件及分析器不可转换为区块。包含分析模式组件的组或者表单组同样不可转为区块。                                                                                                                       |
* 产生变化的操作

| 操作       | 说明                                                                                                                    |
| ---- | ---- |
| 删除组件     | 在删除分析模式组件时，分析器会同步被删除。![删除组件及分析器](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/0937681361/p327702.jpg) |
| 添加或删除子组件 | 在非分析模式的地图内创建分析模式子组件时，会提示是否需要转为分析模式并添加分析器。当分析模式地图内的最后一个分析模式子组件被删除时，会提示是否需要退出分析模式并删除分析器。                                |

## BI分析组件系统自动化配置说明

在您配置组件交互过程中，系统会自动进行一些必要的功能配置修改以减少您的操作量，提高使用的流畅度。目前可进行自动化配置的操作包括：
* 添加字段后自动打开对应视觉通道的映射（颜色、大小和形状映射等）。
* 删除字段后自动关闭对应视觉通道的映射（恢复固定值）。
* 添加字段后自动选择聚合方式。
* 自动选择地理角色（经度、纬度、adcode）。

## 对原分析组件的数据兼容性支持

画布中已添加的原分析组件，如果已经配置数据内容可以继续展示，但是无法修改数据。如果要修改数据内容，可转为分析模式，原字段配置将会保留。

您可以在原分析组件的数据面板 \> 配置数据源处查看相关说明。![配置分析组件兼容性支持](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/0937681361/p326067.jpg)
