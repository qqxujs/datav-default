本文介绍空间构建平台内上传自制的精模建筑模型时的质量标准及模型优化建议。

## 精模建筑模型质量标准

| 质量标准 | 说明                                                                                                                                     |
| ---- | ---- |
| 几何规范 | 模型比例与真实世界近似，DataV中三维场景使用的单位是米，不同的建模软件单位不一样，比如blender的建模单位是米，maya的建模单位是厘米。 单个模型中心点（几何中心或者底面中心）位于建模软件坐标原点。 假如是多模型构成的场景模型，场景中心位于建模软件坐标原点。 |
| 材质贴图 | 建议采用PBR金属度-粗糙度工作流的贴图。 贴图尺寸最好在2的次幂，比如1024\*1024。 贴图总数尽量少，有美术能力的可以参考PBR6张贴图设置。 单张贴图大小不超过2M。                                              |
| 压缩方式 | 较大的模型采用draco压缩方式。                                                                                                                      |
| 模型大小 | 压缩包小于100M。 大量相同静态对象（几何相同+材质相同）采用实例化建模。                                                                                                 |

## 精模建筑模型优化建议

仅针对需要展开的精模建筑模型，本案例以Blender为例。 

| 优化建议        | 说明                                                                                                                                                                                                                          |
| ---- | ---- |
| 明确的命名规范     | 能明确并识别模型展开的内容，如floor\_01等。![Blender建模示例](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6296612561/p437720.jpg)                                                                                               |
| 三维内容组织      | 以Blender为例： 为了方便获取统一的楼层ID，建议每层的根结点是个空对象，如：Shift+A->Empty->Plain Axes。 将楼层内mesh均关联到对应楼层ID的空对象上。 ![建模示例](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6296612561/p437733.jpg)                                 |
| 实例化对象       | 根据对应的楼层进行分层实例化。例如：若1楼和2楼要进行展开，那么1楼和2楼中相同的桌子需要进行分开实例化建模。                                                                                                                                                                     |
| 模型缩放属性建议归一化 | 以Blender为例： 图 1\. 归一化前 ![归一化前](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6296612561/p437736.png) 图 2\. 归一化后 ![归一化后](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6296612561/p437737.png) |
